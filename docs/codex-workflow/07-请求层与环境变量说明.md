## 环境变量设计

### 当前环境文件

- `.env.development`
- `.env.test`
- `.env.production`
- `.env.analyze`
- `.env.example`

### 环境变量说明

| 变量名 | 说明 | 示例 |
| --- | --- | --- |
| `VITE_APP_TITLE` | 前端应用标题 | `Sy Admin` |
| `VITE_API_BASE_URL` | Axios `baseURL` | `/` |
| `VITE_PROXY_TARGET` | 开发代理目标地址 | `http://127.0.0.1:8001` |
| `VITE_USE_PROXY` | 是否启用 Vite 代理 | `true` |
| `VITE_ENABLE_AUTH_GUARD` | 是否开启路由鉴权守卫 | `true` / `false` |
| `VITE_USE_VISUALIZER` | 是否开启打包体积分析 | `true` / `false` |
| `VITE_HTTP_TIMEOUT` | 请求超时时间，单位毫秒 | `15000` |
| `VITE_API_SUCCESS_CODE` | 统一成功业务码 | `1000` |
| `VITE_TOKEN_HEADER_KEY` | token 请求头名称 | `Authorization` |
| `VITE_ENABLE_HTTP_ERROR_MESSAGE` | 是否开启全局错误提示 | `true` |
| `VITE_LOGIN_PATH` | 登录页路径 | `/login` |

### 环境差异

#### 1. development

- 默认开启 Vite 代理
- 默认关闭路由鉴权守卫
- 默认超时 `15000`

用途：

- 本地联调和壳子开发

#### 2. test

- 默认开启路由鉴权守卫
- 保留测试环境构建入口
- 默认超时 `20000`

用途：

- 测试环境构建
- 提前验证“接近生产”的请求行为

#### 3. production

- 默认关闭 Vite 代理
- 默认开启路由鉴权守卫
- 默认超时 `20000`

用途：

- 生产构建
- 线上部署

#### 4. analyze

- 只用于打包分析
- 默认开启 `VITE_USE_VISUALIZER=true`

## Vite 中的环境变量读取方式

### 1. 构建配置层

`vite.config.mjs` 使用：

```js
loadEnv(mode, process.cwd(), '')
```

用途：

- 读取当前 mode 对应的 `.env.*`
- 控制代理开关
- 控制是否启用 `rollup-plugin-visualizer`

### 2. 运行时配置层

`src/config/index.js` 负责统一读取 `import.meta.env` 并做类型转换。

当前对外导出：

- `appConfig`
- `requestConfig`
- `isHttpSuccessCode`

这样做的目的：

- 避免在业务代码里散落读取 `import.meta.env`
- 集中管理布尔值、数字值的解析逻辑

## 请求层目录结构

```text
src
└─ http
   ├─ client.js
   ├─ index.js
   ├─ interceptors.js
   ├─ request.js
   ├─ status.js
   └─ utils.js
```

### 文件职责

#### `client.js`

- 创建 axios 实例
- 注入 `baseURL`
- 注入 `timeout`
- 初始化拦截器

#### `interceptors.js`

- 请求拦截器
- 响应拦截器
- token 自动注入
- HTTP 状态处理
- 业务码处理
- 登录失效跳转

#### `request.js`

- 对外统一请求入口
- 封装 `get/post/put/patch/delete`
- 封装 `text/upload/download/downloadToFile`

#### `status.js`

- 维护 HTTP 状态提示文案
- 维护业务码默认提示文案
- 维护超时/取消请求判断

#### `utils.js`

- 判断是否为标准响应包
- 构造业务错误对象
- `FormData` 构造能力
- 下载文件名解析
- Blob 下载触发

#### `index.js`

- 统一导出 `http`
- 统一导出 `request`
- 统一导出上传/下载辅助方法

## 请求行为约定

### 基础请求配置

- `baseURL`：来自 `VITE_API_BASE_URL`
- `timeout`：来自 `VITE_HTTP_TIMEOUT`
- 默认 `Content-Type`：`application/json`
- 成功业务码：`1000`
- token 请求头：`Authorization`

### token 注入规则

与 Step 1 保持一致：

- 头名称：`Authorization`
- 值：直接传原始 token
- 不拼 `Bearer `

默认策略：

- 请求默认自动带 token
- 开放接口显式传 `withToken: false`

当前已按此规则处理的模块：

- `src/api/modules/auth.js`
- `src/api/modules/app.js`

## 响应处理策略

### 1. 标准成功响应

后端标准结构：

```json
{
  "code": 1000,
  "message": "success",
  "data": {}
}
```

当前前端处理：

- `code === 1000` 时默认返回 `data`
- 需要完整响应包时，可传 `returnFullResponseData: true`

### 2. 业务码统一处理

当前已统一处理：

- `1000`：放行
- `1001`：业务失败
- `1002`：参数校验失败
- `1003`：服务处理失败

当前策略：

- 非成功业务码统一弹错误提示
- 同时抛出 `BusinessError`
- 便于后续业务层按需补充精细处理

### 3. HTTP 状态统一处理

当前已统一处理：

- `401`
- `403`
- `404`
- `408`
- `500`
- `502`
- `503`
- `504`

重点策略：

#### `401`

- 清空本地登录态
- 默认跳转登录页
- 带上 `redirect` 回跳参数

说明：

- 当前没有实现“自动刷新 token 队列”
- 该能力留到真实认证接入步骤统一完成

#### `403`

- 统一提示当前账号无访问权限

#### `500`

- 统一提示服务异常

#### 超时

- 统一提示请求超时

#### 网络异常

- 统一提示检查后端服务或代理配置

### 4. 原始文本返回

为了兼容 Step 1 已确认的 HTML 接口，当前请求层支持：

- `request.text(...)`

适用场景：

- `/admin/base/open/html`
- `/admin/base/sys/param/html`

处理方式：

- `responseType: 'text'`
- 不走默认 JSON 业务码解包

## 上传与下载预留能力

### 上传

已提供：

- `request.upload(url, payload, config)`
- `createFormData(payload, fileFieldName)`

默认约定：

- 自动使用 `multipart/form-data`
- 默认文件字段名建议仍使用 `file`

与 Step 1 保持一致：

- 上传接口：`/admin/base/comm/upload`
- 成功后 `data` 直接返回文件 URL 字符串

### 下载

已提供：

- `request.download(url, config)`
- `request.downloadToFile(url, config)`
- `resolveFileNameFromHeaders(headers)`
- `downloadBlobFile(blob, fileName)`

当前说明：

- 后端项目里尚未发现统一下载接口
- 这部分先预留基础能力
- 等后续真实下载场景出现时再补充文件名、鉴权和预览策略

## 请求方法扩展约定

当前 `request` 支持的扩展配置：

| 字段 | 作用 | 默认值 |
| --- | --- | --- |
| `withToken` | 是否自动注入 token | `true` |
| `showErrorMessage` | 是否显示全局错误提示 | `true` |
| `skipAuthRedirect` | 401 时是否跳过登录跳转 | `false` |
| `returnFullResponseData` | 成功时是否返回完整响应包 | `false` |
| `returnRawResponse` | 是否直接返回 axios 原始响应 | `false` |
| `fileName` | 下载时的兜底文件名 | `download` |

## 前端接口模块目录规范

当前目录：

```text
src
└─ api
   ├─ index.js
   └─ modules
      ├─ app.js
      └─ auth.js
```

后续扩展规范：

### 1. 一个后端领域一个模块文件

建议继续按后端业务域拆分：

- `auth.js`
- `app.js`
- `user.js`
- `role.js`
- `department.js`
- `dict.js`
- `upload.js`
- `space.js`

### 2. API 模块只做“接口函数定义”

模块里只保留：

- 请求路径
- 请求方法
- 参数位置
- 与请求层相关的特殊配置

不要放入：

- 页面状态
- 组件逻辑
- 表单联动逻辑
- 菜单树转换逻辑

### 3. 开放接口要显式声明

例如：

- `withToken: false`
- `skipAuthRedirect: true`

这样做的目的：

- 避免调用开放接口时错误带 token
- 避免登录前接口触发无意义跳转

### 4. 特殊返回类型要显式声明

例如：

- HTML 内容：`request.text`
- 上传：`request.upload`
- 下载：`request.download`
