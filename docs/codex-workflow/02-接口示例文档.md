# 接口示例文档

## 文档目标

挑选首批最适合前端先开发的接口，按“前端直接可用”的格式给出参数与返回示例。

## 确认状态说明

- `已确认`
- `基于代码推断`
- `尚未确认`

## 1. 获取验证码

- 接口名称：获取验证码
- 模块归类：认证
- 请求方式：`GET`
- 请求路径：`/admin/base/open/captcha`
- 是否需要 token：否 `已确认`
- token 请求头：无 `已确认`
- path 参数：无 `已确认`

### Query 参数

| 字段 | 类型 | 是否必填 | 说明 | 确认状态 |
| --- | --- | --- | --- | --- |
| `width` | number | 否 | 验证码宽度 | 已确认 |
| `height` | number | 否 | 验证码高度 | 已确认 |
| `color` | string | 否 | 验证码文字颜色 | 已确认 |

### Body 参数

无 `已确认`

### 成功返回结构

```json
{
  "code": 1000,
  "message": "success",
  "data": {
    "captchaId": "3c9b8f8a-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "data": "data:image/svg+xml;base64,PHN2Zy..."
  }
}
```

### data 字段说明

| 字段 | 类型 | 说明 | 确认状态 |
| --- | --- | --- | --- |
| `captchaId` | string | 验证码 ID，登录时要回传 | 已确认 |
| `data` | string | base64 图片 | 已确认 |

### 前端调用注意事项

- 建议页面加载时先调用一次
- 验证码错误后应重新刷新图片
- `data` 可直接绑定到图片地址

## 2. 登录

- 接口名称：后台登录
- 模块归类：认证
- 请求方式：`POST`
- 请求路径：`/admin/base/open/login`
- 是否需要 token：否 `已确认`
- token 请求头：无 `已确认`
- query 参数：无 `已确认`
- path 参数：无 `已确认`

### Body 参数

| 字段 | 类型 | 是否必填 | 说明 | 确认状态 |
| --- | --- | --- | --- | --- |
| `username` | string | 是 | 用户名 | 已确认 |
| `password` | string | 是 | 密码 | 已确认 |
| `captchaId` | string | 是 | 验证码 ID | 已确认 |
| `verifyCode` | string | 是 | 验证码内容 | 基于代码推断 |

说明：

- DTO 声明里 `verifyCode` 是 `number`
- 服务层实际按字符串验证码比较
- 前端建议统一按 `string` 提交

### 成功返回结构

```json
{
  "code": 1000,
  "message": "success",
  "data": {
    "expire": 7200,
    "token": "eyJhbGciOi...",
    "refreshExpire": 1296000,
    "refreshToken": "eyJhbGciOi..."
  }
}
```

### data 字段说明

| 字段 | 类型 | 说明 | 确认状态 |
| --- | --- | --- | --- |
| `expire` | number | token 过期秒数 | 已确认 |
| `token` | string | 访问 token | 已确认 |
| `refreshExpire` | number | refreshToken 过期秒数 | 已确认 |
| `refreshToken` | string | 刷新 token | 已确认 |

### 前端调用注意事项

- 登录成功后至少要存 `token` 和 `refreshToken`
- 访问接口时把 `token` 放进 `Authorization`
- 不要拼 `Bearer `

## 11. 文件上传

- 接口名称：文件上传
- 模块归类：基础能力
- 请求方式：`POST`
- 请求路径：`/admin/base/comm/upload`
- 是否需要 token：是 `已确认`
- token 请求头：`Authorization` `已确认`
- query 参数：无 `已确认`
- path 参数：无 `已确认`
- 请求类型：`multipart/form-data` `已确认`

### FormData 字段

| 字段 | 类型 | 是否必填 | 说明 | 确认状态 |
| --- | --- | --- | --- | --- |
| `file` | File | 是 | 上传文件 | 基于代码推断 |
| `key` | string | 否 | 指定相对路径或文件名 | 已确认 |

说明：

- 后端直接读取第一个上传文件
- 字段名在代码里不是强约束的命名常量
- 前端建议统一使用 `file`

### 成功返回结构

```json
{
  "code": 1000,
  "message": "success",
  "data": "http://127.0.0.1:8001/upload/20260415/xxxx.png"
}
```

### 前端调用注意事项

- `data` 直接是 URL 字符串
- 当前默认上传大小限制是 `200mb`
- `key` 不能传绝对路径、`..`、`./` 这类非法路径
