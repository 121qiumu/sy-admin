# 前端接口总览

## 文档目标

这份文档只从前端开发视角整理后端项目中实际暴露的后台接口，强调“前端能直接怎么用”。

## 确认状态说明

- `已确认`：可直接从控制器、鉴权、中间件、框架源码中确认
- `基于代码推断`：由实体、服务查询字段、统一 CRUD 规则推断
- `尚未确认`：代码里没有充分信息，联调时需要再次确认

## 路由与鉴权总规则

- 管理后台接口前缀：`/admin/...` `已确认`
- token 请求头：`Authorization` `已确认`
- token 格式：直接放原始 token，不加 `Bearer` `已确认`
- path 参数：当前管理后台接口未发现 `/:id` 这类 path 参数风格，统一按 query 或 body 传参 `已确认`
