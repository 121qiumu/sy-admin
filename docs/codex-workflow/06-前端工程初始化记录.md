## 关键配置说明

### 1. Vite

- 使用 `vite.config.mjs`
- 配置了 `@ -> src`
- 开发服务端口：`5173`
- 已配置代理：
  - `/admin` -> `VITE_PROXY_TARGET`
  - `/upload` -> `VITE_PROXY_TARGET`

### 2. 环境变量

- `.env.development`
  - 默认 `VITE_API_BASE_URL=/`
  - 默认 `VITE_PROXY_TARGET=http://127.0.0.1:8001`
  - 默认 `VITE_ENABLE_AUTH_GUARD=false`
- `.env.production`
  - 默认 `VITE_ENABLE_AUTH_GUARD=true`
- `.env.analyze`
  - 默认 `VITE_USE_VISUALIZER=true`

### 3. Router

当前仅保留工程骨架所需的最小路由：

- `/login`
- `/dashboard`
- `404`

说明：

- 当前是静态壳子路由
- 已预留守卫逻辑
- 后续步骤再接入真实登录态和权限菜单

### 4. Pinia

  - `refreshToken`

### 5. Axios

## 当前已知取舍

### 1. 开发环境默认关闭鉴权守卫

原因：

- 当前步骤只做工程初始化
- 真实登录、验证码、token 刷新和权限菜单会在后续步骤接入
- 先保证开发时可以直接预览后台壳子

### 2. 生产环境默认开启鉴权守卫

原因：

- 避免把“初始化阶段的开发便利策略”直接带到生产配置
