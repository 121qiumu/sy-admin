# 前端对接约定

## 1. 确认状态图例

- `已确认`：源码能直接确认
- `基于代码推断`：由框架约定、实体字段、查询逻辑推断
- `尚未确认`：运行时行为需要联调验证

## 2. 基础请求约定

### 2.1 接口前缀

- 管理后台接口统一从 `/admin` 开始 `已确认`

### 2.2 Token 规则

- 请求头名称：`Authorization` `已确认`
- header 示例：

```http
Authorization: eyJhbGciOi...
```

- 不要加 `Bearer ` `已确认`

### 2.3 content-type 规则

- 普通接口：`application/json` `已确认`
- 上传接口：`multipart/form-data` `已确认`

## 3. 统一响应规范

### 3.1 标准响应

```json
{
  "code": 1000,
  "message": "success",
  "data": {}
}
```

确认状态：`已确认`

### 3.2 code 约定

| code | 含义 | 确认状态 |
| --- | --- | --- |
| `1000` | 成功 | 已确认 |
| `1001` | 通用业务失败 | 已确认 |
| `1002` | 参数校验失败 | 已确认 |
| `1003` | 核心失败 | 已确认 |

### 3.3 HTTP 状态前端处理建议

| HTTP 状态 | 前端处理建议 | 确认状态 |
| --- | --- | --- |
| `200` | 继续看业务 `code` | 已确认 |
| `401` | 视为登录失效，尝试刷新 token 或跳登录 | 已确认 |
| `403` | 视为无权限，展示无权限提示 | 已确认 |

说明：

- 管理后台权限中间件会抛出 `401/403`
- 所以前端不能只看业务 `code`

## 7. 文件上传下载规则

### 7.1 上传接口

- 上传路径：`/admin/base/comm/upload` `已确认`
- 请求类型：`multipart/form-data` `已确认`
- 成功后 `data` 直接返回文件 URL 字符串 `已确认`
- 当前上传大小限制：`200mb` `已确认`

### 7.2 上传参数约定

| 字段 | 类型 | 说明 | 确认状态 |
| --- | --- | --- | --- |
| `file` | File | 上传文件，建议统一字段名 | 基于代码推断 |
| `key` | string | 指定文件名或相对路径 | 已确认 |

### 7.3 路径安全限制

`key` 不要传：

- 绝对路径
- `..`
- `./`
- `.\`
- 以 `/` 开头的路径

确认状态：`已确认`

### 7.4 下载规则

- 项目中未发现专门的文件下载接口 `已确认`
- 当前按“使用上传返回的 URL 直接访问/下载”处理 `已确认`
- 下载时是否有统一鉴权、文件名响应头、预览策略：`尚未确认`

## 8. 错误码与异常处理规则

### 8.1 前端统一拦截建议

建议 Axios 响应拦截器按下面顺序处理：

1. 先处理 HTTP 状态码
2. 再处理业务 `code`
3. `code !== 1000` 时统一提示并 reject

### 8.2 建议处理策略

| 场景 | 前端建议 | 确认状态 |
| --- | --- | --- |
| `401` | 尝试刷新 token；失败则清登录态并跳登录 | 已确认 |
| `403` | 展示无权限页或无权限提示 | 已确认 |
| `1001` | 直接提示后端 message | 已确认 |
| `1002` | 标记为表单校验或参数错误 | 已确认 |
| 原始 HTML 返回 | 不走默认 JSON 解包逻辑 | 已确认 |

## 9. 前端调用注意事项

### 9.1 已确认事项

- `verifyCode` 前端应按字符串处理
- `permmenu` 返回平铺菜单，需要前端组树
- `/admin/base/comm/upload` 返回字符串 URL
- `/admin/base/open/html` 和 `/admin/base/sys/param/html` 返回原始 HTML
- 用户编辑角色回填以 `/admin/base/sys/user/info` 的 `roleIdList` 为准

### 9.2 基于代码推断事项

- 大多数时间字段会是 `YYYY-MM-DD HH:mm:ss` 字符串
- 非特殊声明的 CRUD `list/page` 返回字段基本与实体字段一致
- 上传表单建议统一使用 `file` 作为文件字段名

### 9.3 尚未确认事项

- 任务 `once/start/stop` 接口的最终返回体细节
- 线上部署时上传 URL 的最终域名
- 是否存在额外的全局异常包装格式
