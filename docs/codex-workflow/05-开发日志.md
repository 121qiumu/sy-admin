### 已确认信息

- 管理后台统一前缀：`/admin`
- token 请求头：`Authorization`
- token 不使用 `Bearer` 前缀
- 标准成功 code：`1000`
- 通用分页结构：`list + pagination`
- 默认分页大小：`15`
- 上传接口为 `multipart/form-data`
- 上传成功后 `data` 返回 URL 字符串

### 基于代码推断的信息

- 大多数时间字段会以字符串形式返回
- 一般 CRUD 列表字段会与实体字段基本一致
- 上传文件建议统一使用 `file` 字段名

### 尚未确认信息

- 任务 `once/start/stop` 的最终响应体细节
- 线上部署时上传 URL 的最终域名
- 是否存在额外统一异常包装

### 对前端最重要的提醒

- `verifyCode` 建议按字符串处理
- `permmenu` 返回的是平铺菜单数组，前端要自己组树
- `html` 类接口返回原始文本，不能按 JSON 解包
- 用户编辑页的角色回填要用用户详情接口的 `roleIdList`
