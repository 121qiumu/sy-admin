## 布局方案说明

### 1. 主布局 `AdminLayout`

本步将后台壳子正式抽出为 `src/layout/AdminLayout.vue`。

当前结构：

- 左侧：菜单区域
- 右侧：顶部导航 + 主内容区
- 桌面端：固定侧栏
- 移动端：抽屉式菜单

设计目的：

- 让布局职责单独收敛
- 避免路由壳子和具体页面耦合
- 为后续 CRUD 页面提供稳定承接容器

### 2. 左侧菜单

左侧菜单继续复用 Step 5 的权限菜单能力，但在视觉和交互上做了布局层升级：

- 保留品牌区
- 补充菜单分组提示文案
- 补充底部说明区
- 调整激活态、悬浮态与圆角风格
- 在移动端改为抽屉中渲染

说明：

- 本步没有改动菜单权限数据来源
- 菜单仍然来自 `/admin/base/comm/permmenu`

### 3. 顶部导航栏

顶部导航栏当前包含：

- 侧栏开关按钮
- 当前页面标题
- 面包屑
- 主题切换入口
- 用户下拉菜单

布局目标：

- 把“全局操作”集中到顶栏
- 让后续页面尽量只关注内容区

### 4. 面包屑

本步新增 `AppBreadcrumb.vue`。

当前生成策略：

- 工作台作为固定首入口
- 动态权限页面优先使用 `route.meta.breadcrumbTitles`
- 静态页面退回到 `route.matched` 的标题链路

这样可以兼容：

- 工作台
- Step 5 的动态权限页面
- 后续新增的真实业务页面

### 5. 内容区与 `AppMain`

`AppMain` 继续承接 `RouterView`，但本步统一了：

- 主内容区的最小高度
- 与顶栏的空间关系
- 页面级容器的承接位置

这样可以保证：

- 页面切换后内容区高度稳定
- 错误页、占位页、真实业务页都能落在统一区域

## 页面容器规范

### 1. `PageContainer` 组件

新增 `src/components/PageContainer.vue` 作为页面容器基础组件。

当前支持：

- `eyebrow`
- `title`
- `description`
- `meta` 插槽
- `actions` / `extra` 插槽

推荐用途：

- CRUD 列表页头
- 详情页头
- 分析页头
- 权限占位页和工作台等概览页

### 2. 基础样式规范

本步在 `src/styles/theme.less` 中补充了通用样式类：

- `page-card`
- `page-section-card`
- `page-grid-two`
- `page-toolbar`
- `page-toolbar__actions`

作用：

- 避免每个业务页从零写布局骨架
- 让页面风格保持一致

## 主题变量设计

### 1. 变量扩展范围

本步对 `src/styles/variables.css` 做了扩展，补齐：

- 布局尺寸变量
- 圆角变量
- 阴影变量
- 表层变量
- 控件背景变量
- 菜单激活态与悬浮态变量
- 品牌色与辅助色变量
- header / sidebar 专属背景变量

### 2. 当前变量特点

主题变量不写死在单个组件中，而是统一落在全局层：

- `:root`
- `html.dark`

这样后续做主题扩展时：

- 可以继续追加新主题
- 可以逐步把更多 Element Plus 令牌映射到全局变量

### 3. light / dark 切换入口

切换入口当前放在顶部导航栏。

切换链路：

1. 顶栏点击切换按钮
2. 更新 `Pinia appStore.theme`
3. 写入本地存储
4. 调用 `applyTheme`
5. 切换 `html.dark` 与 `data-theme`

这和之前登录页的主题方案保持一致。

## 响应式处理

### 1. 布局状态管理

本步在 `app` store 中新增：

- `isMobile`
- `sidebarDrawerVisible`

当前策略：

- 宽度小于 `1080px` 时进入移动端布局
- 移动端侧栏改为抽屉
- 桌面端仍使用可折叠侧栏

### 2. 响应式目标

本步优先保证：

- 顶栏在窄屏下仍可用
- 侧栏在移动端可正常打开关闭
- 页面容器头部能自然折行
- 工作台与权限占位页在中小屏下不炸布局
