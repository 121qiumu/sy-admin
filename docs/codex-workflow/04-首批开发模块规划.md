# 首批开发模块规划

## 规划原则

- 只选最适合管理后台首批落地的模块
- 优先解决“登录 -> 获取用户信息 -> 获取菜单权限 -> 基础系统管理”的闭环
- 不把演示模块、代码生成辅助接口放入首批开发

## 首批推荐模块

| 优先级 | 模块 | 目标能力 | 核心接口 | 依赖关系 | 确认状态 |
| --- | --- | --- | --- | --- | --- |

| 开发辅助接口 | 非终端用户后台业务功能 | 已确认 |

## 推荐开发顺序

### 阶段 A：先打底

1. 登录页
2. token 存储与刷新策略
3. 当前用户信息获取
4. 菜单与权限初始化
5. 基础布局页

### 阶段 B：系统管理最小闭环

1. 部门管理
2. 角色管理
3. 用户管理

理由：

- 用户页依赖角色和部门
- 角色页又依赖菜单和部门

### 阶段 C：基础支撑能力

1. 字典
2. 文件上传
3. 文件空间
4. 参数管理

### 阶段 D：补充管理能力

1. 菜单管理
2. 日志管理

## 每个模块建议达到的最小交付标准

| 模块 | 最小交付标准 | 确认状态 |
| --- | --- | --- |
| 登录认证 | 能登录、能刷新 token、能退出 | 已确认 |
| 当前用户与权限 | 能生成动态菜单、能做按钮权限判断 | 已确认 |
| 用户管理 | 能查、能增、能改、能删、能分配角色、能移动部门 | 已确认 |
| 角色管理 | 能查、能增、能改、能删、能配置菜单与部门权限 | 已确认 |
| 部门管理 | 能树形展示、能排序、能增删改 | 已确认 |
| 字典 | 能维护字典类型和字典项，能给表单下拉供数 | 已确认 |
| 文件上传 | 能上传并返回 URL | 已确认 |
